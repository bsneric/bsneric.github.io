记住下面几个概念：

  • 批量梯度下降：如果我们用训练数据集中所有的数据（n=N）来计算损失值，我们就是在执行一个批量梯度下降
  • 随机梯度下降：如果我们只使用训练数据集中的一个数据（n=1）进行计算，这就是一个随机梯度下降

第5步 清洗数据并重复

现在我们使用更新后的参数值再回到第1步重复执行整个过程，经过多个迭代的执行，就是我们说的模型训练。

迭代的定义

每当训练集（N）中的每个点都已用于所有步骤时，迭代就完成了：正向传递、计算损失、计算梯度和更新参数。

在一个迭代，我们至少执行一次更新，但不超过N次更新。

更新次数（N/n）将取决于所使用的梯度下降类型：

  1. 对于批量（n = N）梯度下降，这是微不足道的，因为它使用所有点来计算损失——一个迭代与一个更新相同。
  2. 对于随机（n = 1）梯度下降，一个迭代意味着N个更新，因为每个数据点都用于执行更新。
  3. 对于小批量（大小为n），一个迭代有N/n更新，因为使用n个数据点的迷你批量来执行更新。

在使用Numpy时，你可以通过arrray创建一个三维数组，是吧。技术上我们可以把它成为“张量”。

注意几个概念：

模型

在 PyTorch 中，一个模型就是一个继承了 Module 类的普通 Python 类。如果你熟悉面向对象编程，你就会知道一个类包含了构造函数、方法、属性等等。现在让我们编写一个 Pytorch 的模型类。

  • 首先是__init__(self)方法，它是 Python 的构造函数，接收类初始化的参数，在我们的例子里参数是 b 和 w。当然，参数的数量可能不止这么多，它可以是一个包含很多参数的类对象。记得在__init__方法中嗲用 super().__init__，这样是为了执行基类 nn.Module 中的初始化逻辑。
  • 接下来要定义 forward(self,x)方法：它负责执行预测值的计算逻辑，也即，给定一个 x 值，输出计算的预测值。记住，你不用直接调用这个方法，你只要调用 model(x)就会自动执行这个方法，它实际上是一个钩子函数。因为，当你调用模型的时候，它不仅执行了 forward 方法，还会调用 backward 方法。
