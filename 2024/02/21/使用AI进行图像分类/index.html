形状(NCHW vs NHWC)

这些首字母缩略词代表什么？

其实它们非常简单：

  • N：代表图像的数量
  • C：代表每个图像的通道数量
  • H：代表每个图像的高度
  • W：代表每个图像的宽度

因此这些缩写的含义如下：

  • NCHW:图像的数量，通道数量，高度，宽度
  • NHWC:图像的数量，高度，宽度，通道数量

基本上，每个人都同意第一个应该是图像的数量，高度和宽度是不可分割的一对。所有这些都归结于通道的数量：对于每个单独的图像，它可能是第一个维度（在HW之前）或最后一个维度（在HW之后）。

关于哪种格式更好、更快或其他什么，有无休止的讨论。我们不会深入。尽管如此，我们需要解决这种差异，因为它是混淆和错误的常见来源，因为每个软件包或框架都使用不同的约定：

  • PyTorch 使用 NCHW
  • TensorFlow 使用 NHWC
  • PIL 使用 HWC

Torchvision

Torchvision 是一个包，它包含了常用的数据集，模型架构和计算机视觉领域所需要的图像转换。

数据集

Torchvision 中包含了大量通用的开箱可用的数据集，比如 MNIST，ImageNet，CIFAR等等。这些数据集都继承自 Dataset类，因此它们可以直接使用 DataLoader 加载。

其中有一个数据集我们需要特别关注，那就是 ImageFolder。它本身不是一个数据集，而是一个可以和你的图像数据一起使用的通用数据集，使用它可以将你的图像组织到一系列的子文件夹中，每个子文件夹可以按照类别命名，其中存放相应的图像数据。

模型

Torchvision 中还包含了一些常用的模型架构，包括它们的预训练权重，这可以用于处理类似图像分类，语义分割，对象识别，实例分割，人像关键点识别以及视频分类等任务。

在这些模型架构中，我们可以找到知名的 AlexNet，VGG（以及它的不同版本 VGG11,VGG13,VGG16,VGG19），ResNet（以及它的不同版本 ResNet18, ResNet34, ResNet50, ResNet101, ResNet152）以及 Inception V3。

在后面的文章中，我们将加载一个预训练模型并进行微调，来适应我们的特殊任务。换句话说，我们将使用迁移学习。

为什么我需要在我的训练图像上做这些操作？

这就是所谓的数据增强。在不收集更多数据的情况下扩展数据集（增强数据集）是一种聪明的做法。通常，深度学习对数据的需求量很大，需要巨量的数据来进行训练。但是搜集大量数据通常有难，有时候甚至是不可能的。

回到数据增强：旋转一个图像并假装它是一个新的图像，或者是将同一个图像进行反转。或者在训练过程中随机的作这些操作，这样模型基于可以看到这个图像的不同版本。

假设我们有一张小狗的图像。如果我们旋转它，它仍然是一个小狗，但是是从不同角度看到的。相比于使用两个图像，我们用数据增强方法，利用已有的一张图像模拟了它的不同角度的样子。这虽然和提供一个小狗不同视角的不同图像不同，但是已经非常接近了，它可以提升我们模型的性能。不用说，数据增强并不适合每项任务：如果你要进行一个对象识别的任务，比如识别一个对象在图像中的位置，你不应该修改它的位置，比如反转、裁剪。增加一些噪声倒是和可以。

这是关于数据增强的一个该要介绍，目的是了解在训练过程中这些转换背后的目的。

还有测试期间的数据增强，它被用于提升模型不熟后的性能。这是一个更高级的话题，以后再讲。

归一化转换

我们现试试最常用的转换：Normalize()。在它的文档中，关于这个转换的介绍是这样的：

使用均值和标准差操作来归一化一个张量。首先进行均值计算：(mean[1],…,mean[n])然后求一个通道上的数据标准差：(std[1],..,std[n])，这个转换将这种将使输入Torch 的张量在每个通道上归一化。比如：output[channel] = (input[channel] - mean[channel]) / std[channel]

还记得我们之前讲过的 StandardScaler 吗？它用于将特征数据标准化。这里的 Normalize 是一个张量版本的标准化过程，它可以在图像的每个通道上将数据进行标准化。

那么为什么叫归一化？

不幸的是，这个归一化的过程有很多名称，涉及首先减去平均值，然后将结果除以标准差。我的观点是，它应该被称为“标准化”，就像在 Scikit-Learn 中那样。因为归一化其实有其他的含义（转换功能，使每个数据点都有一个单位规范）。但是在很多场景中，比如 Torchvision 中，标准化的过程就被称为归一化。

无论名字叫标准化还是归一化，这个转换都会修改一个特征值或一组特征值。居然我们之前讲过的，规范特征值将提升梯度下降的效率。更进一步，就如我们马上会看到的，在训练神经网络时，最好是具有对称值范围的特征值（比如从 -1 到 1）。

根据规定，像素值只能是正值，通常在[0,255]之间。我们看到我们的图像张量的值在[0,1]之间，并且我们只有一个通道。我们可以使用 Normalize()方法来将它的值映射到一个对称的区域。

但是，在计算平均值和标准差之前，我们先将均值甚至为 0.5，把标准差也设置为 0.5。

等等，为什么要这样做？

通过这种方式，我们将有效地执行一个“最小-最大缩放”，它能输出[-1,1]之间的结果。如果我们计算原始范围[0, 1]极端的结果值，就知道为什么要这样做了。

$$
input=0\Rightarrow\frac{0-mean}{std}=\frac{0-0.5}{0.5}=-1 \
input=0\Rightarrow\frac{1-mean}{std}=\frac{1-0.5}{0.5}=1
$$

最终：结果范围是[-1，1]。实际上，我们可以把它设置为我们想要的任何东西。如果我们选择0.25的标准差，我们会得到一个[-2，2]范围。如果我们选择了一个不同于原始范围中点的平均值，我们将最终获得一个不对称范围。

现在，如果我们不厌其烦地计算训练数据的真实均值和标准差，我们就会实现实际标准化；也就是说，我们的训练数据将具有零均值和单位标准差。

SubsetRandomSampler

之前，在创建一个训练数据集的数据加载器时，我们设置了它的 suffle 参数为 True（因为大多数情况下，打乱数据可以提升梯度下降的效率）。有一个非常方便的方法来打乱数据，那就是使用 RandomSampler 的实现类。每次有一个新的小批量数据请求时，它随机抽样了一些索引，并返回了与这些索引对应的数据点。

即使不涉及打乱数据，就像用于验证集的数据加载器一样，也使用了SequentialSampler。在这个例子中，不管是否有一个新的小批量数据请求，这个采样器仅按顺序返回索引序列，并返回与这些索引对应的数据点。

简而言之，采样器可用于返回用于数据加载的索引序列。在上述两个示例中，每个采样器将以数据集为参数。但并非所有采样器都是这样的。

WeightedRandomSampler

在学习二元交叉损失的时候，我们已经谈到了不平衡数据集。我们调整了正向类中数据的的损失权重来弥补数据的失衡。不过，它并不像预期的加权平均。现在，我们可以使用另一种不同的方式来处理这种不平衡：权重采样器。

理论几乎相同，但是我们使用权重进行采样，而不是加权损失：数据点较少的类（少数类）应获得较大的权重，而数据点较多的类（多数类）应获得较小的权重 。 这样，平均而言，我们最终会得到每个类中包含大致相同数量的数据点的小批量：平衡的数据集。

深度模型

就这样，我们在模型中添加两个而不是一个隐藏层，并使其变得更深。 我们仍然从 nn.Flatten 层开始，模型的最后一部分仍然是 Sigmoid，但在已经存在的输出层之前有两个额外的 Linear 层。如下图所示：

顺便说一句，在上图中，w和z的下标表示层和单元从零开始的索引：例如，在输出层中，w20表示对应于层和单元的第一个单元（#0）的权重。 第三层（#2）。
这里发生了什么事？ 让我们来计算一下前向传输； 即从输入 (x) 到输出 (y) 的路径：

  1. 一个图像被扁平化为一个包含 25 个特征的张量，从 x0到 x24（并没有在上图中显示）。
  2. 这 25 个特征值被传输给#0隐藏层的五个单元。
  3. #0隐藏层的每个单元使用从w00 到 w04 的权重值，然后特征值通过权重计算出从 z00 到 z04的输出值。
  4. #0隐藏层的输出值（z00 到 z04）传输给#1隐藏层的三个单元，换句话说，0#隐藏层的输出值是#1隐藏层的输入特征值。
  5. #1隐藏层的每个单元使用从 w10到 w12 的权重和前一层的 z0相应的输出值：z10 到 z12。
  6. #1隐藏层继续将z10 到 z12 的输出值传递给输出层的一个单元，也即#1隐藏层的输出被当作了输出层的特征值。
  7. 在输出层的单元利用它的权重 w20 和前一层的z1 计算出最终的输出 z2。
  8. z2是 logit，它是使用 sigmod 函数转换后的概率。

这里有一系列要注意的事情：

  • 在隐藏层中的所有单元，和在输出层中的单元，处理一系列的输入(x 或 z)并且执行相同的操作（x 或 z 都使用自己的权重），然后输出一个 z。
  • 在隐藏层中，这些操作与我们迄今为止使用的逻辑回归模型完全相同，直到逻辑回归产生logit为止。
  • 将一层的输出视为下一层的特征是完全可以的； 实际上，这是我们将在后续看到的迁移学习技术的核心。
  • 对于二元分类问题，输出层是逻辑回归，其中“特征”是前一个隐藏层产生的输出。

没那么复杂，对吧？ 它实际上看起来像是逻辑回归的自然延伸。 让我们看看它在实践中的表现如何。

激活函数

什么是激活函数？

激活函数是非线性函数。 它们要么压扁直线，要么弯曲直线。 他们将打破深层模型和浅层模型之间的等同性。

你说的压扁和弯曲直线是什么意思？

很好的问题！ 请保持这个想法，我将在后续文章的“特征空间”中对此进行说明。 首先，我们来看看一些常见的激活函数。 PyTorch 有很多激活函数可供选择，但我们只关注其中的五个。

Sigmoid

让我们从最传统的激活函数 sigmoid 开始，我们已经用它来将 logits 转换为概率。 如今，这几乎是它的唯一用途，但在神经网络的早期，人们会发现它无处不在！

$$
\sigma(z)=\frac{1}{1+e^{-z}}
$$

让我们快速回顾一下 sigmoid 的形状：如上图所示，sigmoid 激活函数将其输入值 (z)“压缩”到范围 (0, 1) 中（可以采用相同的范围概率，这就是为什么它 用于二元分类任务的输出层）。还可以验证其梯度峰值仅为 0.25（对于 z = 0），并且当 z 的绝对值达到 5 时它接近于零。

另外，请记住，任何给定层的激活值都是下一层的输入，并且给定 sigmoid 的范围，激活值将以 0.5 为中心，而不是零。 这意味着，即使我们将输入值归一化并传输给第一层，其他层也不会再出现这种情况。

输出是否以零为中心有什么关系?

在前面的章节中，我们标准化了特征（零均值，单位标准差）以提高梯度下降的性能。 同样的道理也适用于此，因为任何给定层的输出都是下一层的输入。 实际上还有更多内容，当我们讨论“内部协变量偏移”时，我们将在 ReLU 激活函数中再次简短地讨论这个主题。

PyTorch 有两种类型的 sigmoid 函数，正如我们在前文中已经看到的：torch.sigmoid() 和 nn.Sigmoid。 第一个是一个简单的函数，第二个是从 nn.Module 继承的成熟类，因此，无论出于何种意图和目的，它本身就是一个模型。

双曲切线（TanH）

双曲正切激活函数是 sigmoid 的演变，因为它的输出是均值为零的值，与其前身不同。

$$
\sigma(z)=\frac{e^z-e^{-z}}{e^z+e^{-z}}
$$

如图 4.12 所示，TanH 激活函数将输入值“压缩”到 (-1, 1) 范围内。 因此，以零为中心，激活值已经（某种程度上）是下一层的归一化输入，使得双曲正切成为比 sigmoid 更好的激活函数。

关于梯度，它具有更大的峰值 1.0（同样，对于 z = 0），但其下降速度甚至更快，接近零到低至 3 的 z 绝对值。 这是梯度消失问题的根本原因，导致网络的训练速度逐渐变慢。
就像 sigmoid 函数一样，双曲正切也有两种形式：torch.tanh() 和 nn.Tanh。

修正线性单元 (ReLU)

也许“挤压”不是正确的方法——如果我们稍微改变一下规则并使用一个激活函数来弯曲（是的，另一个双关语！）这条线会怎么样？ ReLU 就这样诞生了，并衍生出了一整套类似的函数！ ReLU 或其亲戚之一是当今激活函数的常见选择。 它解决了前两个版本中发现的梯度消失问题，同时也是计算梯度最快的。

$$
\sigma(z)=\begin{cases}
z,\ if \ z>0 \
0,\ if \ z<0
\end{cases} \
or \

\sigma(z)=max(0,z)
$$

正如您在图 4.13 中看到的，ReLU 是一个完全不同的野兽：它不会将值“压缩”到一个范围内 - 它只是保留正值并将所有负值变成零。

使用 ReLU 的好处是它的梯度要么是 1（对于正值），要么是 0（对于负值）——不再有梯度消失！

这种模式导致网络更快的收敛。 然而，这种行为也可能导致所谓的“神经元死亡”； 也就是说，输入始终为负的神经元，因此激活值始终为零。 更糟糕的是，负输入的梯度也为零，这意味着权重不会更新。 就像神经元被卡住了一样。

ReLU 的激活值显然不是以零为中心。 它会比双曲正切更糟糕吗？ 当然不是，否则它不会成为从业者中如此流行的激活函数。 尽管 ReLU 的输出不以零为中心，但其梯度相对较大，能够比其他两个激活函数获得更好更快的结果。

对于更深、更复杂的模型，这可能会成为一个通常称为“内部协变量偏移”的问题，这只是因为不同层中的激活值有不同的分布。 一般来说，我们希望所有层都产生具有相似分布的激活值，理想情况下以零为中心且具有单位标准差。
为了解决这个问题，可以使用标准化层，例如nn.BatchNorm。 我们将在后续文章中回顾它。

在 PyTorch 中实现 ReLU 的方法有三种：F.ReLU()、nn.ReLU 和 torch.clamp()。

F 是什么？

F 代表功能，它是 torch.nn.function 的常见缩写（正如我们在本章开头的“导入”部分中所做的那样）。 功能模块有很多……功能，其中许多执行相应模块的操作。 在这种情况下，有一个F.ReLU()，它实际上是由其对应模块nn.ReLU的forward方法调用的。

Leaky ReLU

如何才能让“死亡的神经元”有复活的机会呢？ 如果根本问题是它被卡住了，我们需要稍微推动它。 这就是 Leaky ReLU 的作用：对于负输入，它返回一个微小的激活值并产生一个微小的梯度，而不是两者都固定为零。 负值的乘数 0.01 称为泄漏系数。

$$
\sigma(z)=\begin{cases}
z,\ if \ z>0 \
0.01z,\ if \ z<0
\end{cases} \
or \

\sigma(z)=max(0,z)+0.01 \ min(0,z)
$$

正如您在上图中看到的，Leaky ReLU 与 ReLU 几乎相同，除了左侧有一个几乎看不见的微小斜率。

参数化 ReLU (PReLU)

Parametric ReLU 是 Leaky ReLU 的自然演化：我们不要任意选择泄漏系数（例如 0.01），而是将其设为参数 (a)。 希望该模型能够学习如何防止死亡的神经元，或者如何使它们复活（僵尸神经元？！）。 抛开笑话不谈，这是解决问题的巧妙方法。

$$
\sigma(z)=\begin{cases}
z,\ if \ z>0 \
az,\ if \ z<0
\end{cases} \
or \

\sigma(z)=max(0,z)+a \ min(0,z)
$$

如图 4.15 所示，左侧的斜率现在大得多，准确地说是 0.25，这是 PyTorch 参数 a 的默认值。
